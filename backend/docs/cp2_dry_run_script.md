# CP2 Dry-Run Script

Thirty-minute walkthrough for three reviewers.

1. Open `/verification` and review the per-source pass-rate breakdown. Decide whether the weakest source should be re-run before CP2 approval.
2. Open `/checkpoint-2`. Start in Per-Claim Review and spot-check buyer pain points, intel priorities, and competitive claims. Approve, correct, or remove each item.
3. Switch to Bulk Review. Approve buyer profiles only for accounts where all inferred claims are already reviewed. Remove obvious misfit accounts.
4. Resolve remaining pending claims. CP2 approval should stay disabled until the pending count reaches zero.
5. Read the approval confirmation aloud, approve CP2, then confirm the Phase 4 lock is released in `/pipeline`.
